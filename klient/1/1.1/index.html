"Must have" taggar

Definerar ett html dokuments grundstruktur.

<html>
All html kod skrivs innanför denna tag
<body>
Innehåller majoriteten av koden, bilder, länkar, text osv
<head>
Inom denna tag finns title, information om metadata och länkar till css eller javascript filer
<title>
Sätter hemsidans title som syns i webbläsaren

Begrepp

Doctype
Doctype är en instruktion till webbläsaren om vilken typ av dokument det handlar om. Doctype skrivs överst i ett html dokument (men är ej en html tag).
Teckenkod
Teckenkodningen berättar för webbläsaren vilket språk sidan är skriven på.
Det finns olika "charsets", utf-8 är den vanligaste idag då den klarar av miljontals tecken medans äldre varianter kan leda till att språkspecefika tecken som å,ä,ö inte visas korrekt för användaren.
Språk/lang
Lang attributet används för att deklarera språket texten på sidan och kan användas för att anpassa sidan
om det behövs. T.ex genom att anpassa typsnittet
Validering
Validering av kod görs mot en kodstandard för att se så att koden uppfyller de krav som finns.
Validering är viktigt för att en hemsida ska kunna visas korrekt av webbläsaren.
Ett HTML-dokuments uppbyggdnad
Ett minimalt html dokument ser ut såhär:

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<title>Title</title>
</head>
<body>
</body>
</html>

Man bygger sedan upp dokumentet med hjälp av html-taggar som omsluter sitt innehåll.

Meta information

Metadata är data om data. I ett html dokument syns metadata inte, men kan läsas av webbläsare och sökmotorer för att få information om hemsidan. Meta tagen kan ha olika attribute som i sin tur kan ha varierande värden.

Author
Skaparen av html dokumentet
Description
Innehåller en beskrivning av hemsidan i "ordboksformat" dvs den ska vara lämplig för listning i t.ex sökmotorer.
Generator
Ovanligt då det inte behövs för kod skrivet av en användare i en editor utan är till för att visa vilken mjukvara som genererat sidan
Http-equiv
Erbjuder information om innehållet i dess "content" attribut.
  • Content-type: sätter charset för dokumentet
  • Default-style: definerar dokumentets standard stylesheet
  • Refresh: sätter ett tidsintervall då sidan ska uppdatera sig själv
Keywords
Nyckelord används för att hjälpa sökmotorer att ta reda på vad ett dokument handlar om och därmed hamna högre upp på sökningar. Men pga missbruk (utvecklare radade upp onödigt många) så är de för de flesta onödiga idag då t.ex Google inte använder sig av nyckelorden alls. Däremot gör t.ex den kinesiska sökmotorn Baidu det.
Robots
Innehåller information till sökmotorernas robotar, om sidan ska arkiveras eller inte, om roboten ska fortsätta till sidor länkade i dokumentet med mera.

Text

Det finns många taggar för att formatera text
  • Abbr(håll musen över)
  • Adress används för att visa en adress
    Written by Jon Doe.
    Visit us at:
    Example.com
    Box 564, Disneyland
    USA
  • Code, definerar en snutt datorkod (läs nedan)
  • Blockquote används för att citera en annan sida.
    When you call the activate() method on the robotSnowman object, the eyes glow.
  • Br ger en
    radbrytning i texten
  • Cite Detta är en titel på ett mästervärk (låt, tavla, film etc)
  • Dfn markerar den definerande instansen av en term, vanligtvis den första användningen av termen i ett dokument
  • Div används för att omsluta element (definera en sektion) och sedan formatera dessa med gemensam css för att styra t.ex färger, layout etc
  • Em används för att betona ett ord i en text
  • h1-h6 ger rubriker i olika storlekar från

    h1

    till
    h6
  • Kbd används för att definera tangentbords input för användaren, t.ex "Tryck ctrl+v för att klistra in"
  • P innehåller ett stycke med text, och har

    radbrytning före samt efter

  • Pre omsluter förformatterad text. 
                        Det kan vara output från ett datorprogram och
                    behåller båda mellanslag
                    och radbrytningar
  • Q, till skillnad från blockquote som har citat från andra källor används taggen för kortare citat
    Exempelvis Build a future where people live in harmony with nature.
  • Samp används för att visa exempel på output från ett datorprogram
  • Span innehåller mindre text än hela stycken, och saknar radbrytning före samt efter
  • Strong ger text fetstil

Övriga

A
Används för att länka till andra sidor inom eller utanför sidans domän.
Relativ länk
Använder sig av en sökväg som utgår från nuvarande plats ex /site/page/3
Absolut länk
Använder sig av en absolut sökväg inkl protokoll, t.ex http://foo.com/site/page/3. En länk till en extern sida måste vara absolut
Ankarlänk
Används för att länka till andra html element på sidan med hjälp av dess id som sökväg, t.ex #textrubrik
Bildlänk
En bildlänk är en länk som består av en bild man klickar på.
Protokolllänk
En protokolllänk använder det protokoll den nuvarande sidan är hämtad med
genom att url:en inleds med //. På det sättet behöver man inte veta om sidan är tillgänglig via t.ex https eller http
Mailto-länk
Används för att länka till användarens associerade program för att skriva ett nytt mail till den angivna adressen
Listor

Det finns tre typer av listor. Description list (dl), unordered list (ul) och ordered list (ol). Dl använder sig av dt taggen för att ange en titelbeskrivning och dd för själva beskrivningen. Ol och ul använder sig av li taggen för sitt innehåll och saknar titel för varje item. Den här sidan är ett exempel på en description list, se nedan för exempel på ordnade och oordnade listor.

Oordnad lista (ul)

  • dd
  • ol

  • li
Ordnad lista (ol)

  2. dt
  3. dd
  4. ol
  5. ul
  6. li
Img
Används för att visa en bild
Map och Area
En bildkarta är en bild med klickbara ytor. Area taggen skrivs alltid inom map taggen som kopplas ihop med önskad bild via sitt namn attribut och bildens usemap attribut. 

                        <img src="planets.gif" width="145" height="126" alt="Planets" usemap="#planetmap">
                        <map name="planetmap">
                            <area shape="rect" coords="0,0,82,126" alt="Sun" href="sun.htm">
                            <area shape="circle" coords="90,58,3" alt="Mercury" href="mercur.htm">
                            <area shape="circle" coords="124,58,8" alt="Venus" href="venus.htm">
                        </map>
