De tre olika webbsidelayouterna

En webbsidelayout kan vara fixed, fluid eller elastic. Använd CSS och inte tabeller för att skapa din webbsidelayout. Tabeller är enbart ämnade för att visa tabulär data, förr var det dock vanligt att man använde tabeller för att få ett rutnät att placera ut innehåll i. Det är dock ej korrekt och det går utmärkt att uppnå samma utseende med CSS. Fördelar är bland annat att sidan fungerar bättre för sökmotorer, fungerar bättre (mer konsekvent) i olika webbläsare och klienter. Du får även snabbare laddtider då presentationsdatan inte behöver laddas ner för varje sida (eftersom den finns i din externa CSS-mall). Och ju mindre data din sida kräver destu lägre kostnader får du för bandbredd.

Vilken layout ska man välja?

Det finns inte ett rätt svar, utan de har alla sina för och nackdelar. Planerar du noga och tänker efter före så kan du lyckas skapa en bra, användarvänlig, design som drar nytta av de olika fördelarna. Några saker en utvecklare måste tänka på som skiljer sig mellan besökarna är skärmupplösning, val av webbläsare, hur stor del av skärmen som används åt fliken (är webbläsaren inte maximerad? har användaren extra toolbars?)

Fixed

En fixerad layout har en wrapper, ett element som får en fast bredd. Elementen inuti får sedan antingen fast bredd eller en bredd i %. Det viktiga är att användarna får se samma bredd oavsett upplösning. Fördelar är att det blir enklare att förutse hur det kommer se ut för användarna, designen är enklare att bygga och ändra och man får mindre problem med bilder, videos och liknande som har en fast storlek. Nackdelar är bland annat att mindre skärmar kan kräva en horizontal scrollbar, användare med stora skärmar kan få onödigt tomrum på sidorna och rankas vanligtvis lägre när det pratas om användarvänlighet.

Fluid

I en flytande layout sätts bredden på elementen i % av användarens tillgängliga ytan även om vissa saker, t.ex marginaler kan ha sin storlek i px. En väldesignad flytande layout anpassar sig väl för olika användare och kan ta bort behovet av horizontella scrollbars på mindre skärmar vilket anses störande och inte så visuellt tilltalanade. Det kan däremot vara svårare att upptäcka och förutse problem för designern om hen inte testar på olika skärmupplösningar.

Elastic

En elastisk design är en mix av fixerad och flytande. Här används enheten em för att sätta storleken på element. Em är en enhet styrd av storleken på användarens textstorlek. Eftersom användare kan ha olika preferenser så gör det att sidan anpassas utifrån dessa. Utfört korrekt kan en elastisk layout vara väldigt användarvänlig då elementen kan växa sig större/mindre. Däremot är en elastisk design den svåraste att göra och kräver mer testning för att få allt att stämma.

Begrepp

Position
Styr hur elementet ska placeras, absolut antal pixlar från övre vänstra hörnet, relativt från dess placering eller fixerat så det följer med när du scrollar.
Top, left, bottom, right
Används tillsammans med position för att placera elementet. top:50px; flyttar t.ex ett element 50px neråt (från toppen)
Z-index
Styr vilka element som ska ligga överst/underst. Det styr vad användaren ser ifall två element överlappar varandra. T.ex vill en designer nog att text ska synas framför/över bakgrundsbilden. Z-index fungerar enbart på positionerade element. Det element som ska vara längst fram i förgrunden ska ha högst värde.
Float och clear
Float styr hurvida ett element ska "flyta" åt höger eller vänster. Det innebär att elementet flyttas så långt som möjligt i vald riktning. Element som kommer efter ett flytande element kommer flöda runt detta (t.ex text som flödar på sidan och under en bild). För att stoppa detta, använd clear. Clear styr vilka sidor om ett element andra element ej får flyta. Exempel: 

                        img {
                            clear: left;
                        }
Width, height, max-width, max-height, min-width, min-height
Dessa attribut styr storleken på element. width säger att ett element är X stor medans min-width och max-width säger att ett element inte får vara mindre eller mer än X. Ett element med min-width:100px; kan alltså vara 100px eller större, men ej 99px eller mindre. Tvärtom gäller såklart om man sätter en maximal bredd. Man kan även använda det i media queries för att skapa stilregler som bara gäller på skärmar med en storlek på minst X pixlar osv.
Vertical-align
Styr ett elements placering i vertikal riktning.
